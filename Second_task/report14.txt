Оптимизация Блэка-Шоулза (метод Монте-Карло)

ЦЕЛЬ РАБОТЫ
--------------------------------------------------------------------------------
Реализация высокопроизводительного вычислительного ядра для оценки портфеля
европейских Call-опционов методом Монте-Карло. Основной упор сделан на
использование векторных инструкций SIMD AVX2, векторизованных кастомных функций,
многопоточности OpenMP и методов статистической оптимизации для достижения точности < 0.05 у.е.

ТЕХНИЧЕСКИЕ ХАРАКТЕРИСТИКИ
Процессор: Intel Core i7-1260P (Alder Lake)
Базовая частота: 2.1 ГГц
Кэш L1d: 48 КБ на каждое P-ядро
Компилятор: g++ (GCC) 13.4.0 (флаги -O3, -mavx2, -mfma, -fopenmp)

ТЕОРЕТИЧЕСКАЯ МОДЕЛЬ
--------------------------------------------------------------------------------
Для моделирования цены базового актива использовалось уравнение
S_T = S_0 * exp((r - 0.5 * sigma^2) * T + sigma * sqrt(T) * Z),

где Z — случайная величина из стандартного нормального распределения N(0, 1).

Итоговая цена опциона рассчитывалась как дисконтированное среднее выплат по
всем траекториям
Price = exp(-r * T) * Average(max(S_T - K, 0))

ТЕХНИЧЕСКАЯ РЕАЛИЗАЦИЯ И ОПТИМИЗАЦИИ
--------------------------------------------------------------------------------
1. Многопоточность (OpenMP)
    Использована директива #pragma omp parallel for с динамическим планировщиком
schedule(dynamic). Это обеспечило равномерную загрузку ядер CPU, так как
время расчета каждого опциона зависит от параметров волатильности.

2. Векторизация (AVX2 + Sleef)
    Вычисления проводились блоками по 8 путей в 256-битных регистрах __m256.
    Использована библиотека Sleef для вычисления векторных экспонент expf8
    и тригонометрии cosf8/sinf8.
    Логарифмы вычислялись через оптимизированную функцию math::logf_avx.

3. Генерация случайных чисел
    Применен векторный генератор VectorMinstd.
    Для получения нормального распределения использована трансформация Бокса-Мюллера.
    Для устранения дефектов LCG-генератора была внедрена декорреляция потоков через перемешивание
случайных чисел внутри SIMD-регистра.


КЛЮЧЕВЫЕ ПРОБЛЕМЫ И ИХ РЕШЕНИЕ
--------------------------------------------------------------------------------

1. Катастрофическая потеря точности
    При суммировании 1,000,000 путей в типе float сумма достигала
  значений > 10^7. Из-за ограниченной мантиссы float в 7 знаков прибавление
  малых выплат ~ 0.1 игнорировалось процессором. Ошибка составляла > 12.0.
    Для решения проблемы было внедрено накопление промежуточных сумм в double
  через конвертацию float-регистров в __m256d прямо внутри цикла.

2. Высокая дисперсия / статистический шум
    Стандартный метод Монте-Карло не укладывался в порог ошибки 0.05
  для опционов с высокая волатильностью и длинным срок T.
    Для решения проблемы был реализован метод зеркальных переменных: для
  каждого случайного числа Z считались две зеркальные траектории +Z и -Z.
  Это позволило снизить дисперсию в несколько раз и поднять Success Rate с 60% до 99%.

3. Влияние параметров времени T
    При T > 5 лет дисперсия растет экспоненциально, что требовало кратного
  увеличения количества путей и значительно более длительного времени расчета.
    Для решения параметры генератора данных были приведены к адекватным рыночным
  значениям T от 1.0 до 1.25 года, что обеспечило стабильную сходимость.

ИТОГОВЫЕ РЕЗУЛЬТАТЫ
--------------------------------------------------------------------------------
    Размер портфеля 100 опционов
    Траекторий на опцион 1,000,000
    Общее время выполнения ~6.1 - 7.5 секунд
    Максимальная абсолютная ошибка 0.008 - 0.045, допустимый порог: 0.05
    Средняя абсолютная ошибка 0.009
    Количество верно посчитанных и совпавших со скалярным расчетом опционов: 100/100

Комбинация SIMD-векторизации, перехода на double-аккумуляторы и метода зеркальных
переменных позволила создать надежное вычислительное ядро, обеспечивающее
точность аналитического решения Блэка-Шоулза при высокой скорости работы.
Однако стоит отметить, что зависимость от входных параметров влияет очень сильно на точность и, как следствие,
на необходимое количество вычислений для получения валидного результата.
