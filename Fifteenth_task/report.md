# High-Performance Quantitative Pricing Engine

## **Целевая аудитория:** Разработчики ядра на C/C++, системные архитекторы.

### 1. Описание системы
**HP QPE** это вычислительное ядро для высокочастотной оценки финансовых деривативов и расчета риск-метрик. Архитектура построена по **многоуровневому паттерну Layered Architecture**, что позволяет жестко изолировать платформо-зависимые оптимизации, например, SIMD-инструкции, управление иерархией кэш-памяти, от высокоуровневой финансовой бизнес-логики.

### 2. Структура системы
1.  **Низкоуровневое ядро/HPC Core** базируется на кастомной реализации математических функций в виде библиотеки `MathLib`, векторизованной линейной алгебре и статистически валидированных генераторах псевдослучайных чисел.
2.  **Слой моделирования/Stochastic Layer** предоставляет абстракции для стохастических дифференциальных уравнений (геометрическое броуновское движение) и вычислительные движки (симуляции Монте-Карло, биномиальные деревья).
4.  **Уровень приложения/Application Layer** финансовые калькуляторы: оценка Американских и Европейских опционов, API для расчета чувствительности портфеля/«греков».

### 3. Обоснование выбранных архитектурных решений

*   **Отказ от стандартной `libm` в пользу кастомной `MathLib`**
    *   *Причина:* Стандартные функции, например `std::log`, `std::exp`, вызывают сериализацию потока инструкций и ветвления, что блокирует автовекторизацию и создает вычислительный bottleneck при обходе миллионов узлов биномиального дерева.
    *   *Решение:* Реализован режим Enhanced Performance. Использован метод Table-Lookup с использованием 8-битной индексации мантиссы в связке с аппроксимацией минимаксными полиномами по схеме Эстрина. Это обеспечило 100% утилизацию FMA-портов процессора с математически гарантированной погрешностью $< 3.5$ ULP.
*   **Иерархия памяти в линейной алгебре**
    *   *Причина:* Перемножение матриц ковариации для корзинных опционов является строго Memory-Bound задачей. Наивный построчный доступ приводит к постоянному вытеснению кэш-линий и деградации производительности.
    *   *Решение:* Внедрен Tiling матриц на блоки $64 \times 64$ для постоянного удержания рабочих данных в L1D-кэше. На уровне микро-кернелов применен Register Blocking (для процессоров, поддерживающих AVX2 утилизация 14 из 16 доступных YMM регистров) с использованием явных интринсиков AVX2 и AVX512.
*   **Паттерн распараллеливания RNG**
    *   *Причина:* Использование потоков OpenMP в методе Монте-Карло с единым объектом RNG приводит к race condition у данных. Использование инициализации потоков разными seed-значениями математически не гарантирует отсутствие пересечения последовательностей, те отсутствие независимости.
    *   *Решение:* Имплементирован алгоритм `discard` над ядром генератора MT19937. Это математически гарантирует независимость и непересекаемость потоков данных. Качество энтропии подтверждено встроенной двухуровневым тестированием ($\chi^2$, Колмогоров-Смирнов, автокорреляция) со ссылками на уже существующие батареи тестов и литературу.

### 4. Стратегия тестирования и контроля качества
*   Низкоуровневая библиотека `MathLib` покрывается тестами на всем диапазоне значений `float`. Валидация ULP-ошибок проводится относительно высокоточных эталонов. Для генерации минимаксных полиномов используются инструменты произвольной точности (например Sollya/MPFR).
*   Для бизнес-логики (генерация биномиальных деревьев) применяется интервальное тестирование граничных состояний в зоне Deep In-The-Money с целью контроля и предотвращения эффекта катастрофической потери точности при вычислениях.
